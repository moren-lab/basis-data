Apa itu DDL?

DDL (Data Definition Language) adalah bagian dari bahasa SQL yang digunakan untuk mendefinisikan, membuat, mengubah, dan menghapus struktur atau objek dalam basis data seperti tabel, kolom, indeks, dan relasi antar tabel. Tujuan utama DDL adalah untuk membentuk kerangka atau struktur fisik dari sebuah database agar dapat digunakan untuk menyimpan dan mengelola data dengan teratur. Beberapa perintah utama dalam DDL antara lain CREATE, ALTER, dan DROP. Perintah CREATE digunakan untuk membuat objek baru seperti tabel atau database, ALTER digunakan untuk mengubah struktur tabel yang sudah ada, misalnya menambah atau menghapus kolom, dan DROP digunakan untuk menghapus objek seperti tabel atau database dari sistem. Setiap perintah DDL bersifat otomatis tersimpan permanen di dalam database setelah dijalankan, artinya perubahan yang dilakukan tidak dapat dibatalkan menggunakan perintah seperti ROLLBACK. Dengan demikian, DDL memiliki peran penting dalam tahap perancangan dan pengaturan struktur dasar dari sebuah sistem basis data sebelum data dimasukkan atau dikelola melalui perintah SQL lainnya.
